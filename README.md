# Context Graph Benchmarks

Three real-world benchmarks proving the Enterprise Context Graph architecture
against public datasets. Each benchmark demonstrates a different core capability.

| # | Benchmark | Dataset | What It Proves |
|---|-----------|---------|----------------|
| 1 | **ICIJ Offshore Leaks** | 800K+ entities, multi-party ownership | s-Adjacency finds shell-company clusters in <2s |
| 2 | **SCOTUS Citations** | 30K+ Supreme Court cases | 2-Morphisms map real legal precedent chains |
| 3 | **FDA FAERS** | 20M+ adverse event reports | Hyperedge traversal catches multi-drug interactions |

## How It Works

Each benchmark follows the same pattern:

1. **Domain Config** (`config.yaml`) — defines entity types, roles, and hyperedge structure
2. **Download** (`download.sh`) — fetches the public dataset
3. **Ingest** (`ingest.py`) — parses raw data into domain-agnostic `Hyperedge` objects using the config
4. **Benchmark** (`benchmark.py`) — runs s-adjacency traversal, measures timing, counts clusters/paths

The core engine is **identical across all three**. Only the config and ingest change.
That's the "any vertical" proof.

## What We Measure

| Metric | Description |
|--------|-------------|
| **Cluster count** | s-connected components at IS >= 2 |
| **Noise reduction** | Connections at s=1 vs s=2 (should be ~87% reduction) |
| **Traversal latency** | Time for BFS / Yen's K-paths at various graph sizes |
| **Hub nodes** | High-degree entities (key people, drugs, companies) |
| **2-Morphism chains** | Precedent / override relationships discovered |
| **Comparison** | Same queries on Neo4j (binary) and vector DB (embeddings) |

## Research References

### [Arcgentica — REPL Agents + Recursive Delegation](research/arcgentica_analysis.md)

Symbolica AI achieved **85.28% on ARC-AGI-2** (SotA) with ~350 lines of Python using
REPL-based agents with recursive delegation. Their architecture maps directly to our
multi-agent reasoning layer:

| Arcgentica Concept | Context Graph Equivalent |
|---|---|
| REPL Agent with live objects | ExecutiveAgent reasoning over TypeDB graph objects |
| Recursive delegation (`call_agent()`) | ContextAgent → ExecutiveAgent → GovernanceAgent |
| `soft_accuracy()` feedback loop | 2-morphism feedback loop (extract → store → richer graph) |
| Schema-constrained returns | TypeDB schema enforcement (invalid actions structurally impossible) |

Key takeaway: the agentic wrapper **doubles** weaker model performance (Opus 4.5: 28% → 50%).
The model doesn't get smarter — it gets a better environment to reason in.
Our hypergraph topology IS that better environment.

Full analysis: [`research/arcgentica_analysis.md`](research/arcgentica_analysis.md)
